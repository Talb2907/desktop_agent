# Personal Desktop Agent

## Project overview
An Electron desktop app — a personal AI agent that understands free-text requests
in Hebrew and English and autonomously decides which tools to use.
Built to scale from a simple assistant to a full multi-agent system.

## Tech stack
- Electron (desktop app for Windows)
- React + Vite (UI — built for future complexity)
- Tailwind CSS for styling
- Node.js + JavaScript
- Claude API (claude-sonnet-4-5) with tool use
- dotenv for API key management

## UI language
Both Hebrew and English. RTL support for Hebrew. The agent always responds
in the same language the user writes in.

## Agent architecture — reasoning loop
1. User sends a free-text message
2. Claude decides which tool(s) to call
3. The app executes the tool and returns the result to Claude
4. Claude reasons over the result and either calls another tool or returns a final answer
5. The UI shows the user what the agent is doing at each step

## Current tools (Phase 1)

### search_files
Search for files and folders on the local machine.
- Input: query (string), path (optional, defaults to home dir)
- Output: list of matching file paths with metadata

### read_file
Read the contents of a file.
- Input: filepath (string)
- Output: file contents as text (first 5000 chars)

### search_emails
Search Outlook emails using PowerShell.
- Input: query (string), date_filter (optional: "today" | "week" | "month")
- Output: list of matching emails (subject, sender, date, preview)

### read_calendar
Read upcoming calendar events from Outlook via PowerShell.
- Input: days_ahead (number, default 7)
- Output: list of events (title, date, time, location)

### fetch_webpage
Fetch and extract text content from a URL.
- Input: url (string)
- Output: page title + main text content (first 3000 chars)

## Future phases (do not build yet — architecture must support them)
- Phase 2: Long-term memory with SQLite + embeddings (vector search)
- Phase 3: Write actions — send emails, create files, update calendar
- Phase 4: Multi-agent — specialized agents coordinated by an orchestrator
- Phase 5: Full web browsing with Playwright

## Important implementation notes
- API key must come from .env — never hardcode it
- PowerShell commands must handle errors gracefully (try/catch)
- Always show agent "thinking" steps in the UI — never a blank wait
- The agent loop must support multiple consecutive tool calls before responding
- RTL: wrap Hebrew text in dir="rtl", LTR for English
- Keep agent.js clean and modular — it will grow significantly in Phase 4
